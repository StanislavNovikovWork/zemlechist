# Миграция базы данных для поддержки двух периодов работы

## Обзор
Эта миграция добавляет поддержку двух периодов работы для строительных площадок.

## Что изменяется
- Добавляются новые поля в таблицу `construction_sites`:
  - `duration_period1_start` (DATE) - начало первого периода
  - `duration_period1_end` (DATE) - конец первого периода  
  - `duration_period2_start` (DATE) - начало второго периода (опционально)
  - `duration_period2_end` (DATE) - конец второго периода (опционально)

## Как запустить миграцию

### 1. Запустите скрипт миграции
```bash
cd /Users/stanislav/Documents/it/my-app
node scripts/add-duration-periods.js
```

### 2. Проверьте результат
После выполнения миграции:
- Новые поля будут добавлены в таблицу `construction_sites`
- Существующие данные из поля `duration` будут мигрированы в новые поля
- Старое поле `duration` останется для обратной совместимости

## Структура данных после миграции

### Старый формат (обратная совместимость)
```json
{
  "duration": "01.01.2024-31.01.2024"
}
```

### Новый формат
```json
{
  "duration": {
    "period1": ["01.01.2024", "31.01.2024"],
    "period2": ["15.03.2024", "30.03.2024"] // опционально
  }
}
```

## API изменения

### GET /api/markers
- Автоматически конвертирует старые данные в новый формат
- Поддерживает обе структуры данных

### POST /api/markers
- Принимает данные в новом формате
- Сохраняет в новые поля если они существуют
- Иначе использует старое поле для обратной совместимости

### PUT /api/markers/[id]
- Обновляет данные в новом формате
- Корректно обрабатывает удаление второго периода

## Фронтенд изменения

### Форма создания/редактирования
- Добавлена возможность указать два периода работы
- Второй период можно добавить/удалить кнопкой

### Отображение в OrdersGrid
- Строй площадки отображаются в оба периода
- Цветовая индикация учитывает оба периода

### SupplierView
- Показывает оба периода на отдельных строках

## Откат миграции
Если нужно откатить изменения:
```sql
ALTER TABLE construction_sites 
DROP COLUMN IF EXISTS duration_period1_start,
DROP COLUMN IF EXISTS duration_period1_end,
DROP COLUMN IF EXISTS duration_period2_start,
DROP COLUMN IF EXISTS duration_period2_end;
```

## Важно
- Миграция безопасна - она не удаляет существующие данные
- Сохраняется обратная совместимость со старым форматом
- После миграции рекомендуется протестировать создание/редактирование строй площадок
